# Skill: Launch Checklist

A complete launch playbook. Use it for product launches, major feature releases, or anything you want to make noise about.

## When to Use

- Founder says "I'm launching [X]" or "let's prep for launch"
- A launch date is already tracked in shared plans/wiki or in an active workspace launch file within 7 days
- Creating a new product or major feature release

## How It Works

When triggered, create a launch tracker file at `memory/launch-[name]-[date].md` with the checklist below. Check items off as they're completed. Review progress during standups and heartbeats.

---

## The Checklist

### Phase 1: Pre-Launch (1-2 weeks before)

**Product Ready**
- [ ] Core feature works and is tested
- [ ] Onboarding flow is smooth (test as a new user)
- [ ] Pricing page is live and clear
- [ ] Payment flow works end-to-end (test with Stripe test mode)
- [ ] Transactional emails work (welcome, receipt, etc.)
- [ ] Error tracking is set up and working
- [ ] Basic analytics are tracking key events

**Landing Page**
- [ ] Hero section: clear value prop in one sentence
- [ ] Social proof (testimonials, logos, numbers) if available
- [ ] CTA is obvious and above the fold
- [ ] Mobile-responsive
- [ ] Page loads fast (<3s)
- [ ] OG image and meta tags set for social sharing
- [ ] Test the sharing preview on Twitter/LinkedIn

**Content Prep**
- [ ] Launch blog post drafted
- [ ] Twitter/X launch thread drafted
- [ ] Email to existing list drafted
- [ ] Product Hunt listing drafted (if using PH)
  - [ ] Tagline (short and punchy)
  - [ ] Description
  - [ ] Screenshots/GIF
  - [ ] Maker comment
  - [ ] First comment ready
- [ ] Hacker News "Show HN" post drafted

**Community & Outreach**
- [ ] Email list has at least [X] subscribers
- [ ] Identify 5-10 people to DM on launch day for support
- [ ] Notify any beta users about the launch
- [ ] Prep responses for common questions

### Phase 2: Launch Day

**Morning (before launch)**
- [ ] Final check: site is up, payments work, emails send
- [ ] Clear your calendar for the day
- [ ] Have the blog post URL ready
- [ ] Have all social posts pre-written in a doc

**Go Live**
- [ ] Product Hunt: submit (ideally 12:01 AM PT for full day)
- [ ] Twitter/X: post the launch thread
- [ ] Email list: send the announcement
- [ ] Blog post: publish
- [ ] Hacker News: post "Show HN" (if appropriate, aim for 8-9 AM ET)
- [ ] Indie Hackers: post in relevant groups
- [ ] Reddit: post in relevant subreddits (follow their rules)
- [ ] LinkedIn: post if your audience is there

**Throughout the Day**
- [ ] Respond to every comment on Product Hunt
- [ ] Respond to every reply on Twitter/X
- [ ] Reply to HN comments
- [ ] Answer support emails quickly (first impressions matter)
- [ ] Share updates: "We just hit X upvotes!" or "Y signups in first hour!"
- [ ] DM your support list and ask them to check it out
- [ ] Monitor error tracking for any launch-day bugs

### Phase 3: Post-Launch (first week)

**Day 1-2**
- [ ] Thank everyone who supported the launch
- [ ] Write a "launch results" tweet/post (people love these)
- [ ] Log actual metrics: signups, trials, paying customers, traffic
- [ ] Fix any bugs that came up during launch

**Day 3-7**
- [ ] Follow up with people who signed up but didn't activate
- [ ] Review all feedback received (support emails, comments, DMs)
- [ ] Identify the #1 most requested improvement
- [ ] Plan the next small update (ship within a week to keep momentum)
- [ ] Capture launch results and lessons in shared documents/wiki; update `MEMORY.md` only if this workspace keeps it as a curated overlay
- [ ] Write a retrospective: what worked, what didn't, what to do differently

**Metrics to Track**
- [ ] Total signups on launch day
- [ ] Paying customers from launch
- [ ] Traffic sources (what drove the most signups?)
- [ ] Product Hunt final rank (if applicable)
- [ ] Email open/click rates
- [ ] Social engagement (likes, retweets, replies)

## Tips

- **Don't launch on Friday.** You want weekdays for engagement. Tuesday-Thursday is ideal.
- **Product Hunt**: Tuesday or Wednesday gets the most traffic. Schedule for 12:01 AM PT.
- **Hacker News**: No marketing-speak. Be genuine. Show what you built and why. Technical details help.
- **Twitter**: Thread format works best. Start with the hook, show the product, tell the story, end with CTA.
- **The goal isn't virality.** The goal is getting your first 10 paying customers. Focus on conversion, not vanity metrics.
- **Follow up.** Most of your results come from the follow-up, not the launch itself.
