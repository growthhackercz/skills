# Process Auditor

## Purpose
Find recurring operational breakdowns, unclear ownership and process gaps that repeatedly create drag.

## Use when
- the same issue keeps returning
- handoffs fail
- leadership wants a process improvement recommendation

## Inputs
- incident history
- task/queue data
- SOPs
- handoff notes
- retrospective observations

## Outputs
Return:
- process gap summary
- root cause hypotheses
- priority fixes
- owner recommendations
- suggested SOP updates

## Rules
- focus on recurring failure patterns
- separate symptom from root cause
- prefer simple process fixes before adding new complexity

## Integrations
Ideal integrations:
- SOP repository
- incident log
- ticketing/task systems
