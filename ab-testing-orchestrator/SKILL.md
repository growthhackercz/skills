# AB Testing Orchestrator

## Purpose
Design and track structured experiments so campaign learning compounds instead of being lost.

## Use when
- creative, audience or message uncertainty exists
- paid or lifecycle marketing needs disciplined testing

## Inputs
- campaign goal
- variables to test
- existing baseline performance
- traffic/budget constraints

## Outputs
Return:
- test matrix
- hypothesis per variant
- success metrics
- stop/continue rules
- analysis plan

## Rules
- test one clear hypothesis at a time where possible
- avoid noisy test design
- state when sample size or traffic is too weak

## Integrations
Ideal integrations:
- ad platform data
- email performance data
- analytics dashboards
